Dear team 14,


Here are feedback for your project proposal.


P1. Strong problem framing with clear novelty: per-element modifiability is a well-defined and previously unexplored task that connects directly to documented failure modes of real browser agents and generative UI systems. 

P2.Comprehensive literature review with explicit positioning relative to closest precedents (Graph4GUI, ULDGNN, Web-GNN), including thoughtful integration of lecture material (Mesh CNNs, PointNet, ResNet). 

P3. Rigorous experimental protocol including app-level holdout, hand-labeled gold test set, inter-annotator agreement, and paired bootstrap for significance, paired with a clear headline hypothesis (GCN vs MLP gap). 

C1. The heuristic labeling pipeline risks circularity. If heuristics use ancestor context (e.g., "currency text inside a checkout ancestor"), GCN propagation along containment edges effectively reconstructs the labeling rule, making the GCN vs MLP gap partially tautological. Clarify whether heuristics are restricted to local signals, or run a sanity check training both models on gold labels only.

C2. The three-class taxonomy is conceptually clear but boundary cases are subjective. The proposal mentions inter-annotator agreement but specifies no κ target, fallback plan if agreement is low, or resolution procedure for ambiguous cases during heuristic labeling.

C3. The gold set (~50 screens, ~2K nodes) is small for a three-class problem with imbalance, especially for the rare "canonical" class. Statistical power for the headline GCN vs MLP comparison and per-class metrics may be limited, and no target sample size per class is stated.
